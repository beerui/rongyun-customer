DajiCS IIFE 完整示例

本页用于演示在纯静态 HTML 页面中，通过 <script> 直接引入 window.DajiCS 的完整接入方式。 包含快速接入、商品页咨询、悬浮客服、事件日志、环境切换与调试方式。

1. 快速接入

如果你只想先把 IIFE SDK 接进现有站点，按下面 4 步即可跑通。

步骤 1：准备 SDK 文件 将 daji-cs.iife.js 放到静态资源目录。可从 npm 包复制，或从本仓库构建产物复制。
步骤 2：页面中引入脚本 在 HTML 中加入 <script src="./daji-cs.iife.js"></script>，加载后可直接使用 window.DajiCS。
步骤 3：页面初始化时执行 boot() 只需执行一次 DajiCS.boot({ baseUrl, apiBase })，用于配置客服站点地址与预投接口地址。
步骤 4：在按钮点击时调用 open() 传入用户身份和 supplierId，即可打开客服；如果是商品详情页，再额外传入 card 即可预投商品卡。 
<script src="./daji-cs.iife.js"></script>
<script>
  DajiCS.boot({
    baseUrl: 'https://cs.chinamarket.cn',
    apiBase: 'https://api.chinamarket.cn',
    debug: true,
  })

  document.getElementById('contact-cs').addEventListener('click', function () {
    DajiCS.open({
      userId: 'u_10086',
      userName: '张三',
      supplierId: 'SUPPLIER_001',
      token: 'your-token',
      language: 'zh',
    })
  })
</script>

更完整的接入说明见 [`src/sdk/README.md`](src/sdk/README.md) 与 [`docs/SDK_USAGE.md`](docs/SDK_USAGE.md)。

2. 本地运行方式

方式 A：从本仓库构建产物复制

pnpm build:sdk
copy dist\sdk\iife\daji-cs.iife.js examples\iife-basic\daji-cs.iife.js

方式 B：从 npm 包复制

copy node_modules\@daji\cs-sdk\iife\daji-cs.iife.js examples\iife-basic\daji-cs.iife.js
npx serve examples/iife-basic
# 然后打开控制台输出的本地地址
建议先在浏览器控制台观察 [DajiCS] 调试日志，再结合本页的事件日志一起排查接入问题。

5. 悬浮客服 Launcher 示例

如果不想通过新标签页承载客服，可以挂载右下角气泡按钮，并使用 iframe widget 在当前页面中展开。

使用建议：标签页模式适合“离站咨询”；iframe 模式适合“站内驻留咨询”。若站点希望降低跳转感知，优先选 Launcher。

6. IIFE 常用 API 速查

boot(opts)

初始化 SDK。整页或整站只需调用一次。

open(opts)

打开客服窗口；若包含 card，则会先尝试预投商品卡。

openSafe(opts)

跳过预投逻辑，直接开窗，适合接口异常时兜底。

mountLauncher(opts)

挂载悬浮客服气泡，并用 iframe widget 承载会话。

on / once / off

订阅事件、一次性监听、取消监听，可用于日志、埋点、角标更新。

reset()

重置内部状态，适合切环境或重新初始化时使用。

推荐接入顺序

  1. 先在测试页中接入 daji-cs.iife.js 并确认 DajiCS.boot() 成功。
  2. 先跑通 open() 基础开窗，再补商品卡预投。
  3. 接入 unread:change、message:incoming 做站点角标或提示。
  4. 若希望站内悬浮交互，再接入 mountLauncher()。
  5. 上线前确认 baseUrl、apiBase、allowedOrigins 与环境一致。

常见排查点

  • DajiCS is not defined：通常是脚本路径不对，或 SDK 文件未复制到当前目录。
  • 能打开但身份不生效：检查 token、userId、supplierId 是否正确。
  • iframe 无消息：确认 baseUrl 对应 origin 是否在白名单中。
  • 预投失败：检查 apiBase 网络可达性与 /sendRyMessage 接口状态。
